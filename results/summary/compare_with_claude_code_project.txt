D. Where the counts first start to differ

The counts differ at the metadata acquisition step, before any downstream
filtering.

Codex starting point:
  19,546 unique Assembly Accessions after collapsing the full NCBI taxon 670
  metadata package.

Claude starting point:
  14,379 Assembly Accessions from a narrower query:
    GenBank only
    released after 2019-01-01
    current records observed
    fixed 27-field metadata table

Therefore, the first divergence is not caused by year filtering, BioSample
deduplication, or assembly quality filtering. It starts with different NCBI
query definitions.

The next major divergence occurs during quality handling:
  Codex keeps 7,700 after collection-year >=2019, then 7,633 after geography,
  then 6,061 after BioSample deduplication, with no quality filter yet.

  Claude first applies genome length and contig-count filters, then applies
  strict collection-year >=2019, resulting in 4,798 primary records.


E. Most likely reasons for the final count difference

1. Different initial NCBI query scope
  This is the earliest and largest structural difference.
  Codex downloaded all taxon 670 metadata without release-date or source
  restrictions. Claude queried only GenBank assemblies released after
  2019-01-01.

2. RefSeq/GenBank handling is different
  Codex includes both GenBank and RefSeq, then removes paired duplicates at the
  BioSample level while preferring RefSeq.
  Claude avoids paired RefSeq duplicates by querying GenBank only.

3. Claude applies assembly quality filters before the primary set
  Claude removes records outside 4.8-5.5 Mb and records with >=300 contigs.
  Codex has only checked assembly quality and has not yet applied quality
  thresholds.

4. Geography handling is stricter in Codex
  Codex removes records lacking geo_location_final.
  Claude does not remove unknown geographic records in the available prefilter;
  it retains 38 "(unknown)" primary records.

5. Suppressed records remain in Codex
  Codex still contains 653 suppressed records after BioSample deduplication.
  Claude raw and primary tables contain only current records, though this was
  not explicitly implemented as a script filter.

6. Missing collection date handling differs in bookkeeping
  Codex excludes missing/unparseable collection dates during the year step.
  Claude parks A+B-passing missing-date records in a review pool, outside the
  primary dataset.

Most likely direct explanation for final numbers:
  Claude's final primary count is lower mainly because it starts from a narrower
  GenBank-only, released-after-2019 query and applies genome size and contig
  number filters before the primary output. Codex currently keeps a broader,
  BioSample-deduplicated set and has not yet applied quality or suppression
  filtering.


F. Recommended filtering logic

Recommended approach:
  Use the Codex logic as the backbone, then add explicit final-stage filters
  inspired by the useful parts of Claude's prefilter.

Reasoning:
  1. Codex has the more complete and auditable starting universe.
     It uses the full NCBI taxon 670 metadata package, preserves RefSeq and
     GenBank information, collapses expanded BioSample attributes explicitly,
     and records each filtering step in logs and summaries.

  2. Codex uses the better species confirmation criterion.
     ANI Submitted species == Vibrio parahaemolyticus is safer than exact
     Organism Name matching and avoids wrongly excluding records with strain or
     serotype suffixes.

  3. BioSample deduplication is methodologically cleaner than GenBank-only
     avoidance.
     Prefer one assembly per BioSample and choose the best representation,
     rather than discarding RefSeq at query time.

  4. Claude's quality filters are useful but should be applied transparently
     after priority groups are identified.
     Genome length and contig-count thresholds are reasonable metadata-level
     QC checks, but applying them too early can accidentally remove priority
     genomes such as ST50, ST36, ST3, New Zealand clinical isolates, or local
     environmental isolates before those priority labels are known.

Suggested practical next logic:
  1. Continue from Codex data/processed/04_biosample_deduplicated.tsv.
  2. Explicitly remove or separately flag suppressed records before final
     accession-list generation.
  3. Explicitly flag atypical records and review warnings such as contaminated,
     genome length too large, and unverified source organism.
  4. Apply assembly quality thresholds as a documented QC step, for example:
       genome length 4.8-5.5 Mb or a justified VP-specific range
       contig number <300 or <500 depending on desired strictness
       optional N50 threshold
       CheckM completeness/contamination only where available, because more
       than half of Codex records lack CheckM values
  5. Before discarding any QC-failing record, annotate priority status:
       all ST50
       all ST36 from 2019 onwards
       all ST3
       New Zealand clinical isolates
       local New Zealand environmental isolates
  6. Keep a separate review table for priority records that fail QC, rather
     than silently removing them.
  7. Perform geographic/year balancing only after species confirmation,
     collection-year filtering, geography availability, BioSample deduplication,
     suppression/atypical review, QC, and priority annotation.

Bottom line:
  The Codex project is currently broader and more traceable; the Claude project
  is narrower and more aggressively prefiltered. For a publishable global
  selection workflow, the strongest approach is Codex's complete metadata
  backbone plus explicit, documented QC and priority-retention rules before
  final balancing.
